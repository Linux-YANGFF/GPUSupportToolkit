# GST 日志分析工具 - 产品需求文档

## 1. 产品概述

### 1.1 产品名称
**GST** (GPU Support Toolkit) - GPU 支持工具套件

### 1.2 产品定位
面向 Linux 平台的 GPU 日志分析工具，帮助开发者快速定位图形性能问题。

### 1.3 核心价值
- **大文件处理**: 支持 > 1GB 的日志文件
- **快速检索**: 关键字、时间段多维度检索
- **性能分析**: 找出最耗时的帧和函数调用
- **一键导出**: 检索结果方便分享和二次分析

---

## 2. 用户故事

### 2.1 典型用户
- GPU 驱动/图形栈开发工程师
- 游戏开发者（性能优化）
- 系统集成测试工程师

### 2.2 用户场景

| 场景 | 用户行为 | 期望结果 |
|:---|:---|:---|
| 分析卡顿 | 打开日志，查找帧耗时最高的帧 | 快速定位到问题帧 |
| 定位API调用 | 输入关键字 "glDrawElements" | 查看所有调用位置 |
| 性能回归 | 对比两个版本的帧耗时 | 找出性能差异 |
| Shader调试 | 查看Shader编译和绑定情况 | 定位Shader问题 |

```
├─────────────────────────────────────────────────────────┤
│                                                         │

│                                                         │
│  ┌─────────────────────────────────────────────────┐    │
│  │                    功能入口                       │    │
│  ├───────────┬───────────┬───────────┬─────────────┤    │
│  │  🔍 关键字 │  ⏱️ 时间段 │  📊 帧分析 │  🎨 Shader  │    │
│  │    检索    │   检索    │           │    统计     │    │
│  └───────────┴───────────┴───────────┴─────────────┘    │
│                                                         │
│  [结果展示]  虚拟表格 + 分页 + 导出                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 功能详情

#### F1: 文件打开
| 项目 | 说明 |
|:---|:---|
| 支持格式 | apitrace 日志 (.api.txt), profile 日志 (.prof.txt) |
| 文件大小 | 无限制（> 1GB 正常工作） |
| 操作方式 | 菜单打开 / 拖拽文件 / 快捷键 `Ctrl+O` |
| 进度显示 | 流式读取时显示进度条 |

#### F2: 关键字检索
| 项目 | 说明 |
|:---|:---|
| 输入 | 支持多个关键字（空格分隔，AND 关系） |
| 匹配 | 不区分大小写，支持正则（可选） |
| 结果 | 显示匹配行的行号、内容 |
| 分页 | 每页 1000 条，支持翻页 |
| 导出 | 一键导出当前页 / 全部结果 |

**示例**:
- 输入: `glDrawElements glBufferSubData`
- 输出: 所有同时包含这两个关键字的行

#### F3: 时间段检索
| 项目 | 说明 |
|:---|:---|
| 输入 | 开始时间(us)、结束时间(us) |
| 范围 | 可选择全部帧 / 指定帧号范围 |
| 结果 | 该时间段内的所有 API 调用 |
| 导出 | 支持导出为 CSV |

**示例**:
- 输入: 开始时间 `1000000`, 结束时间 `2000000` (1s ~ 2s)
- 输出: 该时间段内的所有 OpenGL API 调用

#### F4: 帧分析 - 最耗时帧
| 项目 | 说明 |
|:---|:---|
| 统计维度 | 帧总耗时、swapBuffers 耗时、API 调用耗时 |
| 展示 | Top N 帧列表（默认 Top 20，可配置） |
| 排序 | 按帧耗时降序排列 |
| 跳转 | 点击跳转到该帧详情 |

**帧识别规则**:
- 特征行: `swapBuffers: X us` 或 `frame cost Xms`
- 每遇到一次，帧计数器 +1

#### F5: 函数统计
| 项目 | 说明 |
|:---|:---|
| 统计项 | 函数名、调用次数、总耗时、平均耗时 |
| 分组 | 按耗时降序排列 |
| 过滤 | 可按函数名前缀过滤（如 `gl*`, `glut*`） |
| 图表 | 可选显示耗时分布（后续版本） |

#### F6: Shader 统计
| 项目 | 说明 |
|:---|:---|
| 检测 | 识别 `glShaderSource`, `glCompileShader` 等调用 |
| 统计 | Shader 类型（Vertex/Fragment/Geometry）、编译次数、编译耗时 |
| 展示 | Shader 列表 + 详情 |

#### F7: 结果导出
| 项目 | 说明 |
|:---|:---|
| 格式 | TXT / CSV / JSON |
| 内容 | 当前检索结果 / 指定帧详情 / 统计报告 |
| 保存 | 弹出文件保存对话框 |

## 5. 数据格式

### 5.1 输入日志格式

**apitrace 日志示例**:
```
glBindBuffer: count=491, time=588 us
glBindFramebuffer: count=29, time=25377 us
glDrawElements: count=493, time=11214 us
...
libGL: FPS = 8.9
swapBuffers: 3033 us
423 frame cost 109ms
```

**字段说明**:
| 字段 | 示例 | 说明 |
|:---|:---|:---|
| API名 | glBindBuffer | OpenGL API 函数名 |
| count | 491 | 该帧内调用次数 |
| time | 588 us | 总耗时（微秒） |

### 5.2 帧边界识别

帧结束的标志行:
```
swapBuffers: X us
```
或
```
frame cost Xms
```
